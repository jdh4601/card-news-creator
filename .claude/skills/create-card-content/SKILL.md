# create-card-content

인스타그램 카드뉴스용 컨텐츠를 원문에서 자동 생성한다.
입력: `input/{ID}/text.md` 원문 + `input/{ID}/` 폴더의 사진 6장
출력: 1페이지 제목 후보 5개 → 승인 → 2~5페이지 본문 (페이지별 승인)

---

## 입력 소스 (폴더 구조)

프로젝트 루트 기준:

```
cardnewscontent/
├── input/
│   └── {ID}/               ← 세트 폴더 (예: 0001)
│       ├── text.md         ← 원문 텍스트
│       ├── photo1.jpeg     ← 알파벳 순서대로 1~6번 매핑
│       ├── photo2.jpeg
│       └── ...
└── output/
    └── {ID}/               ← 결과물 저장 위치
```

### 파일 읽기 방법

유저가 ID를 주면 (예: "0001번 작업해줘"):

1. **원문 읽기**: `Read` 도구로 `input/{ID}/text.md` 파일 읽기
2. **이미지 목록 확인**: `Bash ls` 로 `input/{ID}/` 에서 `.jpeg`/`.jpg`/`.png` 파일 목록
3. 파일명 알파벳 순서 → 카드 순서 (1번째 파일 = 1페이지 배경)

유저가 ID 없이 실행하면: `ls input/` 로 존재하는 세트 목록 출력 후 선택 요청.

---

## 브랜드 톤 (Brand Voice)

이 스킬이 생성하는 모든 텍스트에 반드시 적용된다.

| 항목 | 규칙 |
|------|------|
| 어미 | `~했어요`, `~이에요`, `~예요`, `~거예요` (존댓말 해요체) |
| 어조 | 차분하고 담담하게. 과장 없이, 설명하듯 |
| 감탄 | 느낌표(`!`) 사용 금지. 마침표(`.`)로 끝내기 |
| 강조 | 수식어 남발 금지. 사실 하나를 선명하게 |
| 문장 길이 | 한 문장 15자 초과 시 줄바꿈 처리 |
| 금지 표현 | "놀랍게도", "충격적인", "믿기 힘든", "역대급" |

**좋은 예:**
> 처음엔 아무것도 몰랐어요.
> 그냥 해보자는 마음으로 시작했죠.

**나쁜 예:**
> 놀랍게도! 그들은 역대급 성과를 이뤄냈습니다!

## 실행 프로세스

### STEP 0: 파일 로드 + 원문 분석

```
1. Read input/{ID}/text.md → 원문 전체 읽기
2. Bash ls input/{ID}/ → 이미지 파일 목록 (알파벳 순, text.md 제외)
3. 원문 분석:
   - 핵심 메시지 1개 (전체를 관통하는 주제)
   - 자연스러운 장면 전환 3~4개 (2~5페이지 구분 기준)
   - 감정선 흐름 (시작 → 중간 → 반전/마무리)
```

### STEP 1: 1페이지 — 썸네일 제목 5개 제안

표지 제목은 카드뉴스 전체를 대표하는 훅(hook)이다.

**제목 작성 규칙:**
- **2~3줄 구조** 필수. 한 줄로 쓰지 않는다
- 줄마다 7~12자 이내 (총 3줄 이내)
- 구조: `[상황/배경]` → `[행동/선택]` → `[이유/결과]` 순서로 긴장감을 쌓는다
- 마지막 줄이 독자의 궁금증을 폭발시키는 훅(hook)이어야 한다
- 브랜드 톤 적용 (해요체 또는 명사형 종결)
- 숫자 사용 가능 (예: "9년이 지나도\n변하지 않은 것")
- 느낌표 금지. 마침표도 쓰지 않는다 (여운을 남기기 위해)

**좋은 예:**
```
대학 졸업 후
취준 대신 창업을
선택한 이유

내가 도망쳤던 곳으로
다시 돌아와
도전하는 이유
```

**나쁜 예:**
```
학교 옥상에서 로봇을 띄웠어요   ← 한 줄, 훅 없음
거절이 두렵지 않아진 이유       ← 배경 없이 결론만
```

**출력 형식:**
```
📋 1페이지 — 썸네일 제목 후보

1.
[1줄]
[2줄]
[3줄]

2.
[1줄]
[2줄]
[3줄]

3.
[1줄]
[2줄]
[3줄]

4.
[1줄]
[2줄]
[3줄]

5.
[1줄]
[2줄]
[3줄]

🖼 배경 사진: input/{ID}/[첫 번째 파일명]

👉 마음에 드는 번호를 골라주세요. (수정도 가능해요)
```

→ 유저 승인(번호 선택 또는 직접 수정) 후 STEP 2로

---

### STEP 2~5: 2~5페이지 — 본문 (페이지별 순차 승인)

각 페이지마다 아래 구조로 생성 후, 유저 승인을 받고 다음 페이지로 넘어간다.

**페이지 구조:**
1. **소제목** (title): 15자 이내, 그 페이지의 핵심 한 줄
2. **본문** (paragraph): 2~3단락, 각 단락 1~2문장
3. **하이라이트** (highlight): 본문에서 가장 강렬한 문장 1개 선택

**본문 단락 규칙:**
- 단락당 최대 2문장
- 한 문장이 15자 초과 시 자연스러운 지점에서 줄바꿈 (`\n` 표시)
- 단락과 단락 사이에 공백이 있다고 가정하고 작성

**하이라이트 규칙:**
- 본문 문장 중 1개를 그대로 선택 (새로 작성 금지)
- 독자가 공유하고 싶어 할 만한 핵심 문장
- 40자 이내

**출력 형식:**
```
📄 [N]페이지 — [소제목]

소제목: [소제목 텍스트]

본문:
[단락 1 - 1~2문장]

[단락 2 - 1~2문장]

[단락 3 - 1~2문장 (선택)]

하이라이트:
"[하이라이트 문장]"

🖼 배경 사진: input/{ID}/[해당 순서 파일명]

👉 이 페이지 괜찮으면 "다음"이라고 해주세요. 수정이 필요하면 알려주세요.
```

→ 유저 "다음" 승인 후 다음 페이지 생성

---

### STEP 6: 전체 컨텐츠 정리

5페이지 모두 승인 완료 후 전체 내용을 한 번에 정리해서 출력한다.

**출력 형식:**
```
✅ 전체 카드뉴스 컨텐츠 확정

━━━━━━━━━━━━━━━━━━
1페이지 (표지)
제목: [확정된 제목]
사진: input/{ID}/[파일명]

━━━━━━━━━━━━━━━━━━
2페이지
소제목: [소제목]
본문:
[단락 1]
[단락 2]
[단락 3]
하이라이트: "[문장]"
사진: input/{ID}/[파일명]

(3~5페이지 동일 형식)
━━━━━━━━━━━━━━━━━━

👉 HTML 파일을 만들까요? (build-card-html 실행)
```

## 컨텐츠 길이 제한표

카드 사이즈는 1080×1350px이다. 텍스트가 넘치면 디자인이 깨진다.

| 요소 | 최대 길이 | 비고 |
|------|----------|------|
| 1페이지 제목 (H1) | 25자 | 줄바꿈 포함 3줄 이내 |
| 소제목 (title) | 15자 | 1줄 |
| 본문 단락 1개 | 40자 | 줄바꿈 포함 |
| 하이라이트 | 40자 | 1줄 강조 |
| 총 본문 단락 수 | 최대 3개 | 페이지당 |

---

## 흔한 실수와 해결책

| 실수 | 해결책 |
|------|--------|
| ID 없이 시작 | `ls input/` 로 목록 확인 후 유저에게 선택 요청 |
| 문장이 너무 길다 | 15자 기준으로 쪼개서 줄바꿈 추가 |
| 느낌표를 썼다 | 마침표로 교체 |
| 새 하이라이트 문장을 만들었다 | 본문에서 그대로 선택해야 함 |
| 모든 페이지를 한 번에 출력했다 | 페이지별 승인 후 다음으로 |
| 너무 긴 소제목 | 15자 이내로 압축 |
| "놀랍게도" 같은 금지 표현 사용 | 브랜드 톤 규칙 재확인 |
| 썸네일 제목을 한 줄로 썼다 | 반드시 2~3줄 구조로 나눠서 긴장감 유발 |
| text.md 대신 다른 파일을 읽었다 | `input/{ID}/text.md` 경로 확인 |
